## 🎯 核心理念

**两类 Agent，不同使命**：

### 工程开发 Agent（前期重，后期轻）
- 使命：搭建系统、写代码、修 bug
- 特点：一次性任务为主
- 趋势：系统稳定后逐渐减少参与

### 交易决策 Agent（前期轻，后期重）
- 使命：选股、分析、决策、交易
- 特点：持续性任务，越用越聪明
- 趋势：随数据积累和策略优化，作用越来越大

## 📊 交易决策 Agent 架构（7 层漏斗）

```
┌─────────────────────────────────────────────────────────┐
│                    用户指令层                            │
│         "筛选美股科技股" / "对 AAPL 制定交易计划"         │
└─────────────────────────────────────────────────────────┘
                          ↓
┌─────────────────────────────────────────────────────────┐
│  第 1 层：数据获取 Agent (Data Fetcher)                  │
│  - 获取股票基础数据、行情数据、财务数据                   │
│  - 支持：美股 (YFinance) / A 股 (AKShare)                │
└─────────────────────────────────────────────────────────┘
                          ↓
┌─────────────────────────────────────────────────────────┐
│  第 2 层：因子分析 Agent (Factor Analyst)                │
│  - 20 个技术因子：动量、趋势、波动率等                    │
│  - 10 个基本面因子：ROE、PE、负债率等                     │
│  - 输出：因子评分报告                                     │
└─────────────────────────────────────────────────────────┘
                          ↓
┌─────────────────────────────────────────────────────────┐
│  第 3 层：技术分析 Agent (Technical Analyst)             │
│  - K 线形态识别                                          │
│  - 支撑/阻力位分析                                       │
│  - 成交量分析                                            │
│  - 输出：技术面评分 + 关键价位                            │
└─────────────────────────────────────────────────────────┘
                          ↓
┌─────────────────────────────────────────────────────────┐
│  第 4 层：舆论分析 Agent (Sentiment Analyst)             │
│  - 新闻情感分析 (BettaFish + 自研)                       │
│  - 社交媒体情绪监控                                      │
│  - 黑天鹅事件检测                                        │
│  - 输出：舆情评分 + 风险事件列表                          │
└─────────────────────────────────────────────────────────┘
                          ↓
┌─────────────────────────────────────────────────────────┐
│  第 5 层：宏观分析 Agent (Macro Analyst)                 │
│  - 地缘政治分析 (如：中东局势)                            │
│  - 宏观经济数据 (GDP、CPI、利率)                         │
│  - 行业政策分析                                          │
│  - 输出：宏观影响评估 + 风险等级                          │
└─────────────────────────────────────────────────────────┘
                          ↓
┌─────────────────────────────────────────────────────────┐
│  第 6 层：决策 Agent (Decision Maker) ⭐核心             │
│  - 汇总前 5 层所有信息                                    │
│  - 大模型综合判断                                        │
│  - 生成交易方案：买入/卖出/观望 + 仓位 + 价格区间         │
│  - 输出：交易决策报告                                     │
└─────────────────────────────────────────────────────────┘
                          ↓
┌─────────────────────────────────────────────────────────┐
│  第 7 层：风控审核 Agent (Risk Guardian) ⭐把关          │
│  - 规则审核：仓位限制、止损设置等                         │
│  - 大模型审核：逻辑合理性、风险收益比                     │
│  - 一票否决权                                            │
│  - 输出：最终审批结果 + 完整交易报告                      │
└─────────────────────────────────────────────────────────┘
                          ↓
┌─────────────────────────────────────────────────────────┐
│                    执行层                                │
│  - 模拟交易：记录到数据库                                 │
│  - 实盘交易：调用券商 API（未来）                          │
│  - 生成报告：飞书通知 + 网页展示                          │
└─────────────────────────────────────────────────────────┘
```

## 🤖 Agent 详细职责

### 第 1 层：数据获取 Agent

**职责**:
- 接收股票代码，获取完整数据
- 数据源智能切换（美股→YFinance，A 股→AKShare）
- 数据质量检查（缺失值、异常值）
- 缓存管理（避免重复请求）

**输出**:
```json
{
  "symbol": "AAPL",
  "market": "US",
  "price_data": {...},
  "fundamental_data": {...},
  "data_quality": "high",
  "timestamp": "2026-03-02T10:00:00Z"
}
```

**状态**: ✅ 已完成 (`src/data_provider_v2.py`)

---

### 第 2 层：因子分析 Agent

**职责**:
- 计算 20 个技术因子
- 计算 10 个基本面因子
- 因子标准化和评分
- 因子权重动态调整

**输出**:
```json
{
  "symbol": "AAPL",
  "technical_score": 75,
  "fundamental_score": 82,
  "combined_score": 78,
  "top_factors": ["动量强", "ROE 高", "现金流好"],
  "warning_factors": ["估值偏高"]
}
```

**状态**: ✅ 已完成 (`src/factors/factor_analyzer.py`)

---

### 第 3 层：技术分析 Agent

**职责**:
- K 线形态识别（头肩顶、双底等）
- 支撑/阻力位计算
- 趋势线分析
- 成交量形态分析
- MACD/RSI/KDJ 指标解读

**输出**:
```json

  "symbol": "AAPL",
  "trend": "uptrend",
  "support_levels": [170, 165],
  "resistance_levels": [180, 185],
  "pattern": "上升三角形",
  "technical_rating": "买入",
  "confidence": 0.78
}
```

**状态**: ✅ 已完成 (`src/filters/technical_filter.py`)

### 第 6 层：决策 Agent ⭐

**职责**:
- 汇总前 5 层所有分析结果
- 大模型综合判断
- 生成交易方案
- 风险收益比计算
- 仓位建议

**输出**:
```json

  "decision": "买入",

  "holding_period": "2-4 周",

  "risk_reward_ratio": 2.5
}
```

**状态**: 🔄 开发中

---

### 第 7 层：风控审核 Agent ⭐

**职责**:
- 规则审核（硬性风控）
  - 单票仓位≤10%
  - 单日交易≤5 笔
  - 止损必须设置
- 大模型审核（软性风控）
  - 逻辑合理性
  - 风险收益比≥2
  - 无重大风险事件
- 一票否决权

**输出**:
```json
{
  "decision": "批准",
  "original_plan": {...},
  "adjustments": {
    "position_size": "从 15% 降至 10%",
    "reason": "中东局势风险较高"
  },
  "final_plan": {...},
  "risk_notes": ["关注中东局势发展"],
  "approved_by": "Risk Guardian Agent",
  "timestamp": "2026-03-02T10:30:00Z"
}
```

**状态**: ✅ 框架完成 (`src/risk/risk_manager.py`)，需集成大模型

## 💹 交易执行流程

### 场景 1：新建仓位

```
用户指令："筛选美股科技股并交易"
    ↓
1. 选股引擎 → 输出 20 只股票
    ↓
2. 对每只股票执行 7 层分析
    ↓
3. 决策 Agent 生成交易方案
    ↓
4. 风控审核 Agent 审批
    ↓
5. 执行交易（模拟/实盘）
    ↓
6. 生成报告 + 飞书通知
```

### 场景 2：持仓管理

```
用户指令："对 AAPL 制定交易计划"
    ↓
1. 获取持仓数据
    ↓
2. 执行 7 层分析（重点评估是否继续持有）
    ↓
3. 决策 Agent 建议：加仓/减仓/清仓/持有
    ↓
4. 风控审核
    ↓
5. 执行 + 报告
```
